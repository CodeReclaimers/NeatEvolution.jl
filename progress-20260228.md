## Fix Julia 1.10 CI Failures: BoolAttribute `:true` Bug

### Root Cause
In Julia, `:true` and `:false` are `Bool` literals (`true`/`false`), **not** `Symbol("true")` and `Symbol("false")`. The `BoolAttribute` code in `src/attributes.jl` used `:true`/`:false` as if they were Symbols, causing two problems:

1. **Construction:** `default_sym = :true` assigned `Bool(true)`, but the subsequent `Symbol(string(default_sym))` conversion happened to produce `Symbol("true")`, so the stored value was accidentally correct.
2. **Comparison:** `init_value()` compared `attr.default == :true`, which evaluated as `Symbol("true") == true` — always `false`. This caused the function to fall through to `rand(rng, Bool)`, making the `enabled` attribute random instead of deterministic.

### Why It Only Failed on Julia 1.10
`MersenneTwister(42)` produces different random sequences on Julia 1.10 vs 1.11 (due to internal RNG implementation differences). On 1.11, the random `enabled` values happened to be `true`, masking the bug. On 1.10, they came out `false`, disabling all connections and causing the "Feed-Forward Network Pruning" test to fail.

### Fixes
- **`src/attributes.jl`**: Replaced all `:true`, `:false`, `:random` with `Symbol("true")`, `Symbol("false")`, `Symbol("random")` in both the `BoolAttribute` constructor and `init_value()` comparison.
- **`test/runtests.jl`**: The "Add connection mutation" test omitted `node_delete_prob` and `conn_delete_prob`, which defaulted to 0.2 and 0.5 respectively — competing with the 0.9 add probability. On Julia 1.10's RNG sequence, deletions outpaced additions. Added explicit `:node_delete_prob => 0.0, :conn_delete_prob => 0.0` since the test is specifically for add-connection behavior.

### Verification
- Julia 1.10: 431 passed, 0 failed, 4 broken (visualization skips)
- Julia 1.11: 420 passed, 0 failed, 4 broken (visualization skips)

- `src/attributes.jl` — Fixed Symbol literal comparisons
- `test/runtests.jl` — Added explicit zero-probability defaults to connection mutation test
